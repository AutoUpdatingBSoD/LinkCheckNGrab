Why Does LinkCheckNGrab Exist?


This project came out of the need to filter almost 1000 (potentially) dead redirects, by hand if needed.

  • This script is intended to save the user's time, both short and long term.

  • I wouldn't know a link was dead unless I clicked it.

  • The final script saved a lot of time for the team who needed this, as they did not want to check that
    many dead redirects to see which ones were dead.

  • This script greatly reduced that number to a more reasonable value.

How does this project work?

  • You supply the input filename, output filename, URL for the root of your website (if needed, can be ""
    if you don't need it), and redirect prefix (i.e. Redirect 301) in the LinkCheckNGrab.sh file.

  • There's already different versions of LinkCheckNGrab to handle external redirects and regex paths
    (see the GitHub repository under 'Where?')

  • Make sure each redirect is in the format of the configuration input filename (Assuming you're using internal redirects, no regex).

  • Then, just make the file executable with chmod +x LinkCheckNGrab.sh, and run in a Linux environment with ./LinkCheckNGrab.sh
